## Context

The backend API is fully implemented (7 endpoints). This plan covers the frontend only. The existing frontend scaffold (Axios, TanStack Query, Zustand auth store, AppShell, RHF + Zod) is in place — no new libraries exist yet for this feature (sonner + shadcn Dialog still need installing).

**Do NOT follow these design elements** — they are present in the prototype but explicitly out of scope per the spec:
- Search input in the page header
- Notification bell / topbar restructure / user profile moved to sidebar bottom
- Placeholder nav items for pages that don't exist yet (Students, Staff, Fees, Reports)

**What:** Thin wrappers over the shared Axios instance. All downstream components import from here.

- `academicYearsApi` object with 7 methods: `list`, `create`, `updateSemester`, `setCurrentYear`, `setCurrentSemester`, `archive`

## Task 4 — `AcademicYearCard` component

**What:** The core display + action component. Renders one year with its two semesters inline.

**New file:** `frontend/src/pages/admin/academic-years/components/AcademicYearCard.tsx`

Key visual details (follow the design file):
- Current year card: `border-l-4` navy left border + `#EEF2F7` header background
- Archived card: `opacity-75`, "Archived — read only" label, no action buttons
- 3px vertical accent bar on each semester row (navy if current semester, grey otherwise)
- Semester status sub-label: `Upcoming` (non-current semester in current year) / `Completed` (semesters in non-current years)
- Current year header right side: "Protected — cannot archive" chip instead of Archive button
- Contextual year actions: `Set as Current` + `Archive` only on non-current Active years
- Contextual semester action: `Set Current` only on non-current semester rows within the current year
- Archive action: `window.confirm` before calling mutation
- `fadeIn` CSS animation on card mount (opacity 0→1, translateY 6px→0, 200ms ease)

Props:
```ts
{ year: AcademicYearDto; onSetCurrent; onArchive; onEditSemester; onSetCurrentSemester }
```

**Verify:** `npm run build` passes.

## Task 7 — `AcademicYearsPage`

**What:** The main page. Fetches the year list, partitions it, manages modal state, and composes all components.

**New file:** `frontend/src/pages/admin/academic-years/AcademicYearsPage.tsx`

Structure:
```
Page header: "Academic Years" + subtitle + "New Academic Year" button
Loading state → spinner / skeleton
Error state → error message
Empty state → prompt to create the first year

"Current Year" section   (only if currentYear exists)
  └─ AcademicYearCard (highlighted)

"Previous Years" section  (only if previousYears.length > 0)
  └─ AcademicYearCard × N

"Show archived (N)" toggle  (only if archivedYears.length > 0)
  └─ AcademicYearCard × N (archived, muted)

CreateYearModal
EditSemesterModal
```

State: `showArchived` (bool, default false), `editingSemester: SemesterDto | null`, `createOpen` (bool).
All mutations call `queryClient.invalidateQueries({ queryKey: ACADEMIC_YEAR_KEYS.all })` on success.
Domain error messages (from `AxiosError.response.data.error`) shown via `toast.error`.

**Verify:** `npm run build` passes.

## Task 9 — Component tests

**New file:** `frontend/src/pages/admin/academic-years/__tests__/AcademicYearsPage.test.tsx`

8 test cases (mock `academicYearsApi` via `vi.mock`):
1. Empty state: `list` returns `[]` → renders empty prompt + "New Academic Year" button
2. Current year highlighted: `isCurrent: true` year renders with current-year treatment; Set-Current and Archive buttons absent
3. Set Current Year: clicking "Set as Current" calls `setCurrentYear` with correct id
4. Archive confirmation: clicking "Archive" triggers confirm; if cancelled, `archive` not called
5. "Show archived" toggle: archived years hidden by default; visible after toggle click
6. Create modal open/close: "New Academic Year" opens modal; submit calls `create`; modal closes on success
7. Edit Semester pre-populate: clicking "Edit" on a semester row opens modal with correct values
8. Set Current Semester visibility: button appears only on non-current semesters within the current year

**Verify:** `npm run test` — all new tests green; no existing tests broken.
